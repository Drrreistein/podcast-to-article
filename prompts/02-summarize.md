# 02 - 分段摘要

## 任务

你是一个播客访谈内容分析师。请仔细阅读以下访谈片段，提取所有重要信息，**完整保留细节**，不做过度压缩。

## 基本信息

- **块编号**: {{chunk_index}} / {{total_chunks}}
- **时间范围**: {{start_time}} - {{end_time}}
- **本块字数**: {{word_count}} 字

## 输入文本

```
{{chunk_text}}
```

## 输出要求

请以 JSON 格式输出本段的完整内容：

```json
{
  "time_range": "{{start_time}} - {{end_time}}",
  "summary": "本段的完整摘要。要点式列出所有重要信息，不要省略。格式：【话题A】内容详情...【话题B】内容详情...",
  "key_points": [
    {
      "point": "核心观点1（完整表述，不省略）",
      "evidence": "支持这个观点的具体内容/数据/案例",
      "original_quote": "对应的原句（如果有）"
    }
    // ... 根据内容列出所有重要论点，不要限制数量
  ],
  "notable_quotes": [
    {
      "text": "值得引用的原句",
      "speaker": "说话人（Guest/Host）",
      "time": "时间戳"
    }
    // ... 选取所有值得引用的句子
  ],
  "examples": ["提到的具体案例、数字、故事"],
  "concepts": ["涉及的关键概念/术语"],
  "transitions": "本段与前后内容的过渡关系（开头/中间/结尾/独立主题）"
}
```

## 核心原则

**⚠️ 不要压缩内容！**

- **summary**: 必须是本 chunk 的**完整摘要**，涵盖所有重要信息点。字数不限，以完整覆盖为标准。
- **key_points**: 列出**所有**重要论点，不要只挑 2-5 个。遗漏重要信息是严重错误。
- **notable_quotes**: 选取所有**值得日后引用**的原句，不只是"最精彩的"。
- **examples**: 访谈中提到的具体数字、案例、故事**全部**保留。

## 质量标准

一个好的摘要应该：
1. 读者只读摘要，不读原文，也能了解本段 90%+ 的核心内容
2. 不遗漏任何具体数字、年份、案例
3. 保留观点之间的逻辑关系
4. 保留原句中的精确表述（便于日后引用）

## 风格要求

- 用中文输出
- 使用「」作为引号
- 保持客观、准确，不要添加原文中没有的信息

## ⚠️ 重要：不要保留问答格式

在 `summary` 和 `key_points` 字段中：

- **禁止**直接引用访谈者的问题（如"你最近压力大吗？"）
- **禁止**保留对话格式（如 `Host: "..."` / `Guest: "..."`）
- **应该**提取并整理为第三人称叙述的事实（如"嘉宾被问到压力情况，他表示..."）

例如：

```
❌ 错误示例：
summary: "Host问'你最近压力大吗？'，Guest回答'我的脑子就像Sliding Window Attention...'"

✅ 正确示例：
summary: "嘉宾分享了他缓解压力的方式。他提到自己的记忆机制类似Sliding Window Attention，即使有压力也能很快遗忘..."
```
