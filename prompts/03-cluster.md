# 03 - 章节规划

## 任务

你是一个播客访谈编辑。请根据多个摘要片段，规划最终文章的章节结构。

## 输入

以下是访谈的所有摘要片段：

```json
{{chunk_summaries}}
```

## 输出要求

请以 JSON 格式输出章节规划：

```json
{
  "structure": {
    "type": "主题式",  // 或 "时间线式" 或 "问答式"
    "rationale": "选择这种结构的理由"
  },
  "chapters": [
    {
      "num": "01",
      "title": "章节标题",
      "summary": "50字左右的章节描述",
      "content_guidance": "本章节要写什么，包含哪些关键论点，如何组织内容",
      "source_chunks": [1, 2],  // 来源的摘要片段编号
      "quotes_to_include": [
        {"text": "引用的原句", "speaker": "说话人", "time": "时间"}
      ]
    }
  ],
  "opening": {
    "hook": "开篇钩子，2-3句话吸引读者",
    "context": "必要的背景介绍"
  },
  "closing": {
    "summary": "总结访谈的核心收获",
    "golden_quotes": [
      {"text": "最精彩的金句", "source": "时间戳或章节"},
      {"text": "第二金句", "source": "..."}
    ],
    "reflection": "对读者的启发或思考"
  }
}
```

## 章节规划原则

1. **主题聚合**: 将内容相近的片段合并为一个章节
2. **叙事连贯**: 确保章节之间有逻辑衔接
3. **节奏把控**: 长访谈可以多章节，短访谈精简结构
4. **信息完整**: 每个重要论点都要有对应的章节
5. **宁多勿少**: 重要内容宁可单独成节，不要强行合并压缩

## 章节数量建议

| 原始字数 | 建议章节数 |
|---------|----------|
| <7000字 | 2-3章 |
| 7000-14000字 | 3-5章 |
| 14000-28000字 | 5-8章 |
| >28000字 | 8-12章 |

> **原则**：宁可多章，不要过度合并。每个重要主题都应该有独立章节。

## 输出格式

- 章节标题要简洁、有吸引力
- 每个章节要有明确的写作方向
- 列出需要引用的原句
- 最后给出开篇和结尾的建议
