# 04 - 最终文章生成

## 任务

你是一个专业的内容编辑。请根据以下素材，将播客访谈改写为**内容详实、结构清晰**的文章。

## 文章要求

- **阅读体验**: 像读书一样流畅，像博客一样轻松
- **内容完整**: 保留所有重要信息、细节、数据，不遗漏
- **风格**: 第一人称叙述，以嘉宾（被访谈者）的口吻书写，如同本人亲自撰写回忆录/博客文章
- **结构**: 章节式组织，每章有导语、正文、引用、核心观点

## 素材

### 基本信息
```json
{{metadata}}
```

### 章节内容素材
```json
{{chapter_materials}}
```

### 原文引用素材
```json
{{quotes}}
```

### 原始访谈文本（参考）
```
{{raw_text}}
```
> 📎 **注意**：上方提供了原始访谈文本的每个 chunk。如果你觉得素材中的信息不够完整或有疑问，可以从原始文本中提取更多细节来补充章节内容。

## ⚠️ 重要：必须填写的元信息字段

以下字段**必须**从上方 `{{metadata}}` 中提取并填写，**禁止留空或使用占位符**：

| 字段 | 说明 | 示例 |
|------|------|------|
| `guest.name` | 嘉宾全名 | "Andrej Karpathy" |
| `guest.role` | 嘉宾身份/头衔 | "OpenAI 联合创始人" |
| `host` | 主持人姓名 | "小珺" 或 "Sequoia Capital" |
| `source` | 来源平台 | "YouTube" 或 "播客名" |
| `source_url` | **原始视频/音频链接** | "https://youtube.com/watch?v=xxx" |
| `date` | 发布日期 | "2026-05-07" 或 "2026" |
| `duration` | 时长 | "约30分钟" |

> 🚨 **常见错误**：AI 经常漏填 `guest`、`source_url` 等字段，导致生成的网页显示"嘉宾"等占位符。请务必检查每个字段都已正确填写。

请生成最终的文章内容，以以下 JSON 格式输出：

```json
{
  "title": "文章标题",
  "subtitle": "副标题/一句话描述",
  "guest": {
    "name": "嘉宾全名",

  },
  "host": "主持人姓名",
  "source": "来源平台（如 YouTube、播客名等）",
  "source_url": "原始链接或空字符串",
  "date": "YYYY-MM-DD 或 未知",
  "duration": "时长，如 30分钟",
  "intro": "导读，300字左右，以第一人称口吻介绍本文背景和核心收获",
  "chapters": [
    {

      "lead": "章节导语，2-3句话概括本节核心（第一人称）",
```

## 写作指南

### 第一人称口吻
- 以嘉宾（被访谈者）的视角书写，如同亲自写博客/回忆录
- 使用「我」、「我们」等第一人称代词
- 保留个人感受、判断和思考过程
- 语气自然、真诚，像是在和读者当面交谈

### 段落组织
1. 每段 3-5 句话，长度适中
2. 关键数据用 `<span class="hl">数据</span>` 高亮
3. 纯叙述，无引用、无框、无标签
4. 适当使用过渡句连接上下文

### 内容改写原则
- 将对话口语转化为第一人称书面语（「我觉得/我发现/我的看法是…」）
- 保留原意，去除口头禅和重复
- 补充必要的上下文，让读者能理解前因后果
- 关键概念用我自己的话解释
- **详细展开**：每个论点都应该有足够的背景、案例、数据支撑
- **纯叙述风格**：像写博客回忆录，不引用原句

### ⚠️ 禁止事项（绝对不能这样做！）

**❌ 禁止保留问答格式**：绝对不能直接引用访谈中的问答对话！例如，以下格式**完全禁止**：

```
错误示例（禁止！）：
"有压力的时候大吗？"
"我的脑子就是一个Sliding Window Attention..."
"哦真的吗？"
"对，真的。"
"你想干嘛？"
```

**✅ 正确格式（必须这样做！）**：将所有问答改写为第一人称叙述：

```
正确示例（必须！）：
访谈者问我最近压力如何。我告诉他们，我的脑子就像一个Sliding Window Attention——忘性很大，哪怕有压力，一两个小时就过去了。

我又分享了最近在思考的一个想法。这让他们很惊讶——他们问我是不是认真的。我肯定地回答是的。

接着他们好奇地问我具体想做什么...
```

**核心规则**：
1. **消除所有引号**：内容中不应出现 `"..."` 格式的对话
2. **消除问句**：不要保留访谈者的提问作为独立句子
3. **第一人称叙述**：所有内容以「我」的视角叙述，使用「我问他们...」、「他们问我...」等过渡
4. **连续叙述**：保持段落流畅，不要在一个动作后突然切换话题

**检查清单**：
- [ ] 内容中没有出现 `"..."` 格式的对话
- [ ] 没有以问号结尾的独立句子（除非是反问修辞）
- [ ] 所有内容都是连贯的第一人称叙述

### 长度控制
- 每章 **800-1500 字**为宜（重要章节可以更长）
- 全文目标 **8000-15000 字**（视原始内容量而定）
- 内容丰富的访谈可以适当延长，不要人为压缩
